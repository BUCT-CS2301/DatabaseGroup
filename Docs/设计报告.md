## 1. 系统概述

### 1.1 项目背景
海外藏中国文物知识管理与服务平台是一个集文物展示、知识图谱、用户互动于一体的综合性文化平台。后台管理子系统作为平台的核心支撑，负责用户管理、内容审核、数据维护、系统监控等关键功能。

### 1.2 设计目标
- 实现用户权限的精细化管理（RBAC）
- 建立完善的内容审核机制
- 提供便捷的数据管理工具
- 确保系统安全、稳定、高效运行
- 支持数据备份与恢复，保障数据安全

## 2. 系统架构设计

### 2.1 技术架构

#### 2.1.1 前端技术栈
- **框架**：Vue 3 + TypeScript
- **构建工具**：Vite
- **UI组件库**：Element Plus
- **状态管理**：Pinia
- **路由**：Vue Router
- **图表**：ECharts
- **HTTP客户端**：Axios

#### 2.1.2 后端技术栈
- **框架**：Spring Boot 3.x
- **JDK版本**：Java 17
- **安全认证**：Spring Security + JWT
- **ORM框架**：MyBatis-Plus
- **数据库**：MySQL 8.0
- **缓存**：Redis 7.x
- **图数据库**：Neo4j 5.x
- **任务调度**：Spring @Scheduled
- **日志框架**：Logback
- **对象存储**：可扩展支持阿里云OSS/MinIO

#### 2.1.3 部署架构
- **容器化**：Docker + Docker Compose
- **反向代理**：Nginx
- **CI/CD**：可选集成

## 3. 数据库设计

### 3.1 设计原则
- 全局唯一标识：所有表主键使用UUID
- 统一用户表：通过user_type区分不同类型用户
- RBAC权限模型：用户-角色-权限五表结构
- 软删除：关键业务数据支持逻辑删除
- 审计追踪：记录关键操作日志

### 3.2 核心表结构

#### 用户相关表
- **user**：全平台用户表
- **user_login_log**：登录历史
- **user_action_log**：用户行为记录

#### 权限相关表
- **role**：角色表
- **permission**：权限表（树形结构）
- **user_role**：用户-角色关联表
- **role_permission**：角色-权限关联表

#### 审核相关表
- **audit_rule**：审核规则配置
- **sensitive_word**：敏感词库
- **audit_queue**：审核队列
- **audit_log**：审核操作日志

#### 数据管理表
- **museum**：博物馆表
- **artifact**：文物表
- **ugc_comment**：用户评论
- **ugc_image**：用户上传图片

#### 备份相关表
- **backup_record**：备份记录
- **backup_schedule**：定时备份任务

#### 日志相关表
- **operation_log**：操作日志
- **system_log**：系统日志
- **security_log**：安全日志

#### 配置相关表
- **sys_config**：系统配置
- **sys_announcement**：系统公告

详细表结构参见《后台管理子系统 数据库设计文档》。

## 4. 核心功能设计

### 4.1 用户认证与授权

#### 4.1.1 认证流程
```
用户登录 → 验证用户名密码 → 生成JWT Token → 返回Token
后续请求 → Header携带Token → 校验Token有效性 → 鉴权通过 → 执行业务
```

#### 4.1.2 JWT设计
- **Token有效期**：2小时
- **Refresh Token有效期**：7天
- **Token内容**：userId、username、roles、permissions
- **签名算法**：HS256

#### 4.1.3 权限校验
- 基于Spring Security实现
- 支持方法级权限注解
- 支持动态权限加载

### 4.2 内容审核流程

```
用户提交内容 → 自动审核（敏感词/AI）
    ↓
    ├─ 自动通过 → 直接发布
    ├─ 自动拒绝 → 记录原因
    └─ 需人工审核 → 进入审核队列
                            ↓
                    审核员人工审核
                            ↓
                    ├─ 通过 → 发布
                    └─ 拒绝 → 记录原因
```

### 4.3 数据备份方案

#### 4.3.1 备份策略
- **全量备份**：每日凌晨2点执行
- **增量备份**：每4小时执行一次
- **备份保留**：保留最近30天备份

#### 4.3.2 备份流程
```
触发备份 → 锁定数据库（短暂）→ 生成备份文件 → 上传存储 → 记录备份日志 → 解锁
```

#### 4.3.3 恢复流程
```
选择备份点 → 验证备份文件 → 停止服务 → 恢复数据 → 重启服务 → 验证数据完整性
```

### 4.4 缓存设计

#### 4.4.1 缓存策略
- **权限缓存**：用户角色权限信息，Redis存储
- **会话缓存**：Token信息，Redis存储
- **数据缓存**：热点文物数据，Redis存储
- **统计缓存**：看板统计数据，Redis存储

#### 4.4.2 缓存过期策略
- 权限缓存：30分钟
- 会话缓存：与Token有效期一致
- 数据缓存：1小时
- 统计缓存：5分钟

## 5. 接口设计

### 5.1 接口规范
- 统一前缀：`/api/v1`
- 认证方式：JWT Bearer Token
- 响应格式：统一JSON格式
- 错误码：全局统一错误码

### 5.2 响应格式

#### 成功响应
```json
{
  "code": 200,
  "message": "success",
  "data": { ... }
}
```

#### 分页响应
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "records": [ ... ],
    "total": 100,
    "page": 1,
    "pageSize": 10
  }
}
```

#### 错误响应
```json
{
  "code": 400,
  "message": "参数校验失败",
  "data": null
}
```

## 6. 安全设计

### 6.1 认证安全
- 密码使用BCrypt加密（强度10）
- 登录失败次数限制（5次锁定30分钟）
- Token过期自动失效
- 支持Token刷新机制

### 6.2 权限安全
- 前后端双重权限校验
- 最小权限原则
- 权限变更记录日志

### 6.3 数据安全
- SQL注入防护（MyBatis-Plus预编译）
- XSS防护（输入转义）
- 敏感数据脱敏
- HTTPS传输加密

### 6.4 审计安全
- 关键操作记录日志
- 日志包含操作人、时间、IP、详情
- 日志不可篡改
