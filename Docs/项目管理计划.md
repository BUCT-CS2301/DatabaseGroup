## 技术方案与系统架构

### 3.1 总体技术栈

结合团队技术栈熟悉度与系统性能要求，技术选型如下：

| 层次         | 技术选型                                 | 选型依据                                                     |
| :----------- | :--------------------------------------- | :----------------------------------------------------------- |
| **前端**     | Vue 3 + TypeScript + Vite + Element Plus | 组合式API易维护，TypeScript减少运行时错误；Element Plus提供完备的后台表单/表格组件。 |
| **后端**     | Spring Boot 3.x + Java 17                | 生态成熟，事务支持良好，适合权限、日志、备份等企业级功能。   |
| **安全认证** | Spring Security + JWT (或 Sa-Token)      | 实现RBAC、动态权限、操作日志记录                             |
| **数据库**   | MySQL 8.0 + Redis 7.x                    | MySQL存业务数据，Redis做缓存（权限、会话、实时统计）         |
| **图数据库** | Neo4j (引用，非独立建设)                 | 后台通过连接共享的图数据库进行数据查看、编辑与一致性检查。   |
| **对象存储** | 阿里云OSS                                | 存储用户上传的图片、视频及数据库备份文件。                   |
| **任务调度** | Quartz / XXL-JOB 或 Spring @Scheduled    | 定时备份、数据统计、自动审核任务                             |
| **日志监控** | Logback + MySQL                          | 结构化存储操作/安全日志                                      |
| **部署运维** | Docker + Docker Compose                  | 实现前后端一键部署，统一测试环境。                           |

> **架构说明**：子系统复用平台统一的用户体系，用户表可共享但虚拟划分“前台用户”与“后台管理员”，两者登录逻辑及Token生成策略在后端通过标识位字段严格分离。



## 四、项目进度计划 (甘特图概述)

项目开发周期为5周（第11周 - 第15周），分为四个阶段：

### 第一阶段：基础环境与用户认证 (第11周)

- 后端脚手架搭建、数据库设计与建表
- 集成Spring Security与JWT，实现管理员登录及Token刷新
- 前端Vue 3项目初始化，搭建基础布局（Layout）、路由守卫
- **交付**：可运行骨架，完成登录/登出。

### 第二阶段：核心功能模块开发 (第12周)

- **陈翔钰、杨文康**：内容审核模块开发（敏感词库管理、自动审核API、人工审核队列、批量操作）。完成角色管理（增删改查、分配权限）、用户管理（列表、禁用/启用）。
- **陈柯珉、李一川**：数据管理模块（文物列表、字段编辑、CSV导入/导出、图片替换）。
- **交付**：RBAC权限管控生效，审核流跑通。

### 第三阶段：数据运维与系统监控 (第13-14周)

- **李一川**：备份与恢复模块（定时备份策略、手动备份、从备份点恢复）、日志采集与查询API。
- **陈柯珉**：日志管理前端页面、系统监控看板（ECharts实时统计在线人数、新增用户、审核队列积压量）。
- **全员**：数据一致性检查逻辑（关系库与图库同步校验）。
- **交付**：自动化备份机制运行，日志可查，看板数据准确。

### 第四阶段：集成联调、测试与收尾 (第15周)

- 前后端全功能联调，修复Bug
- 编写接口文档与部署手册
- 自动化测试用例编写（权限、审核流）
- 最终验收与汇报准备
