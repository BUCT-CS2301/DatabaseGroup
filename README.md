# 后台管理子系统

## 任务目标

按照教师给出的软件工程题目要求，本小组的任务为：

>主要包括：
>1）角色与权限管理：
>
>- 建立基于角色的访问控制机制，至少定义超级管理员、内容审核员、数据管理员三类角色
>- 支持自定义角色，可灵活分配各模块的细粒度权限
>- 所有权限变更须记录操作日志
>
>2）用户管理：
>
>- 对全平台所有用户进行统一管理，涵盖后台管理员、知识服务子系统用户、掌上博物馆用户
>- 支持用户信息的增删改查、状态管理、行为记录和违规处理（选做）
>
>3）内容审核：
>
>- 建立"自动审核+人工审核"的双重内容审核机制
>- 自动审核包括文本审核、图片审核，且审核结果可配置
>- 人工审核包括待审核队列管理、审核操作、批量审核和审核统计
>
>4） 数据管理：
>
>- 对全平台所有业务数据进行统一管理，包括文物数据、知识图谱、用户生成内容
>- 支持数据一致性检查（选做）
>
>5）数据备份与恢复：
>
>- 保障平台数据安全，防止数据意外丢失
>- 支持手动备份、定时自动备份、备份记录管理、数据恢复和备份存储管理
>
>6）日志管理
>
>- 建立完善的操作日志体系，支持审计与问题排查
>- 包括操作日志、系统日志、安全日志的记录和管理
>- 支持日志查询和导出功能
>
>7）系统监控看板：
>
>- 实时展示平台运行状态，帮助管理员及时发现并处理异常
>- 包括实时数据统计、访问量统计、数据增长统计和异常告警（选做）
>
>8）系统配置管理（选做）：
>
>- 支持在线修改系统运行参数，无需重启服务
>- 包括敏感词库管理、审核策略配置、系统公告管理和功能开关
