## 一、技术栈

| 层次           | 技术选型                                                 | 理由说明                                                  |
| :------------- | :------------------------------------------------------- | :-------------------------------------------------------- |
| **前端框架**   | Vue 3 + TypeScript + Vite                                | 组合式API易于维护，TypeScript增强类型安全，Vite构建速度快 |
| **UI组件库**   | Element Plus 或 Ant Design Vue                           | 成熟的后台组件库，表格、表单、树形权限控制等开箱即用      |
| **状态管理**   | Pinia                                                    | 轻量、类型安全，适合管理用户、权限、全局配置等状态        |
| **HTTP请求**   | Axios + 请求/响应拦截器                                  | 统一处理Token、日志上报、错误提示                         |
| **后端框架**   | Spring Boot 2.7+ / 3.x (Java 17)                         | 成熟稳定，生态丰富，适合权限、日志、备份等企业级功能      |
| **安全框架**   | Spring Security + JWT (或 Sa-Token)                      | 实现RBAC、动态权限、操作日志记录                          |
| **ORM框架**    | MyBatis-Plus                                             | 简化CRUD，支持乐观锁、分页、逻辑删除                      |
| **数据库**     | MySQL 8.0 + Redis 7.x                                    | MySQL存业务数据，Redis做缓存（权限、会话、实时统计）      |
| **任务调度**   | Quartz / XXL-JOB 或 Spring @Scheduled                    | 定时备份、数据统计、自动审核任务                          |
| **文件存储**   | 阿里云OSS / MinIO (自建)                                 | 存储图片（审核）、备份文件                                |
| **日志采集**   | Logback + ELK (可选) 或 直接存MySQL + 定时滚动           | 操作日志、系统日志、安全日志规范化存储                    |
| **监控看板**   | Spring Boot Actuator + Micrometer + Prometheus + Grafana | 实时QPS、JVM、数据库连接等，满足“系统监控看板”要求        |
| **对象存储**   | MinIO                                                    | 备份文件、图片审核缓存                                    |
| **容器化(选)** | Docker + Docker Compose                                  | 便于部署和测试环境统一                                    |
| **版本控制**   | Git + GitLab / GitHub                                    | 团队协作，分支管理                                        |

> 说明：前端也可选用 React + Ant Design Pro，但结合团队可能更熟悉Vue生态，选Vue 3更易上手。后端若团队偏爱Go或Python，也可调整，但Spring Boot在企业级后台中最为稳健。
