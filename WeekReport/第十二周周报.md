### 1.2 后端接口开发进展

本周后端完成了数据管理模块的核心接口实现，具体如下：

#### 文物管理模块（RelicController）

| 接口 | 方法 | 路径 | 功能说明 |
| :--- | :--- | :--- | :--- |
| 分页查询 | GET | /api/v1/data/relics | 支持关键词搜索、博物馆筛选、分页参数 |
| 详情查询 | GET | /api/v1/data/relics/{objectId} | 获取单个文物详细信息 |
| 创建文物 | POST | /api/v1/data/relics | 创建新文物记录，自动生成 imageUrl/imagePath |
| 更新文物 | PUT | /api/v1/data/relics/{objectId} | 支持部分字段更新 |
| 删除文物 | DELETE | /api/v1/data/relics/{objectId} | 软删除，将 isDeleted 置为 1 |
| 上传图片 | POST | /api/v1/data/relics/{objectId}/image | 支持 JPEG/PNG/GIF/WebP，最大 10MB |
| CSV导入 | POST | /api/v1/data/relics/import-csv | 批量导入文物，支持 UTF-8 编码，最大 2000 行 |

#### 博物馆管理模块（MuseumController）

| 接口 | 方法 | 路径 | 功能说明 |
| :--- | :--- | :--- | :--- |
| 分页查询 | GET | /api/v1/data/museums | 支持关键词搜索（name/nameCn） |
| 详情查询 | GET | /api/v1/data/museums/{objectId} | 获取单个博物馆信息 |
| 创建博物馆 | POST | /api/v1/data/museums | 创建新博物馆记录 |
| 更新博物馆 | PUT | /api/v1/data/museums/{objectId} | 支持部分字段更新 |
| 删除博物馆 | DELETE | /api/v1/data/museums/{objectId} | 物理删除 |

#### 核心功能实现细节

**1. 文物图片上传功能**

- 支持格式：JPEG、PNG、GIF、WebP
- 文件大小限制：单文件最大 10MB
- 存储路径：`relics-images/{objectId}.{ext}`
- 格式检测：魔数检测 + ImageIO 二次校验
- 自动清理：上传新图片时自动删除同 ID 其他格式旧图

**2. CSV批量导入功能**

- 编码要求：UTF-8（支持 BOM）
- 文件大小限制：最大 10MB
- 数据行限制：最大 2000 行
- 表头校验：必须包含 CreateRelicRequest 全部 11 个属性列
- 重复检测：自动检测完全相同的数据行
- 事务处理：整批导入失败自动回滚

**3. 权限控制**

- 读操作：任意有效 JWT Token 即可访问
- 写操作：仅 `user_type=ADMIN` 用户可执行
- 使用 `SecurityUtil.requireAdminWritePermission()` 统一校验

#### 路由守卫逻辑

```typescript
router.beforeEach((to, from, next) => {
  const userStore = useUserStore()
  const requiresAuth = to.meta.requiresAuth

  if (requiresAuth && !userStore.token) {
    next('/login')
  } else if (to.path === '/login' && userStore.token) {
    next('/dashboard')
  } else {
    next()
  }
})
```

## 三、技术亮点

1. **CSV导入解析器**：实现了完整的 CSV 解析与校验流程，包括表头规范化（小写、去下划线、去空白）、UTF-8 严格解码、必填字段校验、重复行检测、事务批量插入

2. **图片格式智能检测**：采用魔数检测 + ImageIO 二次校验双重机制，确保上传文件格式正确，支持 JPEG/PNG/GIF/WebP 四种格式

3. **路由守卫机制**：基于路由元信息 `requiresAuth` 实现声明式权限控制，代码简洁易维护

4. **API文档模块化**：将接口文档按业务模块重新组织，提升文档可读性与维护性
